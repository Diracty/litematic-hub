# Litematic Hub в интернет — простая инструкция

Для тех, кто не программист. Делайте **по порядку**, не перескакивайте.

---

## Зачем два сервиса, а не один

Представьте **магазин**:

| Сервис | Что это | Аналогия |
|--------|---------|----------|
| **RelaxDev** | Сайт в браузере + API | Сам магазин, витрина, касса |
| **Supabase** | База данных | Склад, где лежат загруженные файлы |

- **Только Supabase** — есть склад, но **нет сайта**. Не подходит.
- **Только RelaxDev** — есть магазин, но **некуда класть** загруженные `.litematic`. Нужна база.
- **RelaxDev + Supabase** — нормальный бесплатный вариант.

**Supabase не заменяет RelaxDev.** Они работают **вместе**.

---

## Про «лимиты» и «бесплатный старт»

### Replit (то, от чего уходим)

На бесплатном Deploy часто **~30 дней**, потом просят платить. Это вас и бесит — справедливо.

### RelaxDev

«Бесплатный **старт**» = сейчас можно пользоваться **бесплатно** для своих проектов.  
Это **не** обещание «навсегда без ограничений» — у любого хостинга могут быть лимиты по размеру/нагрузке.  
Но это **не те же 30 дней Replit**: обычно проект **живёт**, пока укладывается в бесплатные лимиты платформы.

Точные цифры смотрите на https://relaxdev.ru когда регистрируетесь.

### Supabase (база)

Бесплатный тариф **постоянный** (не «удалят через 30 дней» как у Render).  
Ограничения: объём данных (для вашего проекта обычно хватает), лимиты запросов.  
**24/7** — база в облаке всегда включена, ПК можно выключить.

# Пошагово (4 больших шага)

## Шаг 1. Положить проект на GitHub

GitHub — это «облако для кода», оттуда RelaxDev скачает сайт.

1. Зарегистрируйтесь: https://github.com/signup  
2. Установите **GitHub Desktop**: https://desktop.github.com/  
3. Войдите в GitHub Desktop под своим аккаунтом.  
4. На сайте GitHub: **+** → **New repository** → имя `litematic-hub` → Create (без галочек README).  
5. В GitHub Desktop: **File → Add local repository** → выберите папку:  
   `c:\ProjectsCursor\Lite-Structure-Parser\Lite-Structure-Parser`  
6. Если предложит создать репозиторий — согласитесь.  
7. Внизу напишите комментарий `first upload` → **Commit to main**.  
8. **Publish repository** → выберите `litematic-hub`.

Готово, когда на github.com в репозитории видны папки `artifacts`, `deploy`, `lib`.

## Шаг 2. Создать базу данных (Supabase)

1. Откройте https://supabase.com → **Start your project** → войдите через **GitHub**.  
2. **New project**  
   - Name: `litematic`  
   - Password: придумайте сложный пароль → **запишите в блокнот**  
   - Region: Frankfurt (или любой ближний)  
3. **Create** → подождите 1–2 минуты.

### Скопировать «адрес базы»

1. Слева **Project Settings** (шестерёнка) → **Database**.  
2. **Connection string** → вкладка **URI** → режим **Transaction** (порт 6543).  
3. Скопируйте строку. В ней замените `[YOUR-PASSWORD]` на ваш пароль из блокнота.  
4. Сохраните всю строку в блокнот — это **DATABASE_URL**.  
   Пример вида: `postgresql://postgres.xxxx:ВАШ_ПАРОЛЬ@....supabase.com:6543/postgres`

### Создать таблицы (один раз)

1. В Supabase: **SQL Editor** → **New query**.  
2. На ПК откройте Блокнот → файл:  
   `c:\ProjectsCursor\Lite-Structure-Parser\Lite-Structure-Parser\deploy\init.sql`  
3. Скопируйте **весь** текст → вставьте в SQL Editor → **Run**.  
4. Должно быть зелёное Success.

Если Supabase **не даёт** зарегистрироваться — напишите, дам вариант «всё в Docker на VPS» (сложнее, но без Supabase).

---

## Шаг 3. Запустить сайт (RelaxDev)

1. https://relaxdev.ru → регистрация → **войти через GitHub**.  
2. **Создать проект** / **Deploy** → выбрать репозиторий **litematic-hub**.  
3. Тип: **Docker** (не «авто Next.js»).  
   Подсказка: https://relaxdev.ru/deploy/docker  

Настройки (если спрашивает):

| Поле | Что вписать |
|------|-------------|
| Dockerfile | `deploy/Dockerfile` |
| Контекст / Root | `.` или корень репозитория |
| Порт | `8080` |

4. **Переменные окружения** (Environment / Secrets) — добавить **четыре строки**:

| Имя | Значение |
|-----|----------|
| `PORT` | `8080` |
| `DATABASE_URL` | вся строка из блокнота (шаг 2) |
| `STATIC_DIR` | `/app/public` |
| `NODE_ENV` | `production` |

5. **Deploy** / **Запустить**.  
6. Ждать 5–15 минут. Смотреть **Логи** — в конце что-то вроде «Server listening».

RelaxDev даст **ссылку** на сайт — откройте в браузере. Должна быть страница загрузки `.litematic` (старый интерфейс).

## Шаг 4. Проверка

1. Загрузите файл `.litematic` через сайт.  
2. Должен появиться в списке.  
3. Для загрузчика Minecraft API:  
   - `https://ВАША-ССЫЛКА/api/info/ключ-файла`  
   - `https://ВАША-ССЫЛКА/api/part/ключ-файла/1`

После **обновления парсера** на ПК: снова Commit + Push в GitHub Desktop → RelaxDev пересоберёт → **загрузите `.litematic` заново** (старые данные в базе не обновятся сами).

## Частые вопросы

**Нужен ли включённый компьютер?**  
Нет. Работают серверы RelaxDev и Supabase.

**Платить?**  
На старте — нет, если укладываетесь в бесплатные лимиты. Карту Supabase/RelaxDev могут спросить позже на платных тарифах — на free часто не нужна.

**Почему не Replit?**  
У бесплатного Deploy часто лимит ~30 дней. RelaxDev + Supabase — другая схема.

**Попугай без цвета (Variant)?**  
В JSON части должно быть `"Variant": 0` (или 1–4). Нет — парсер на сервере старый или файл не перезаливали.

**Ничего не понятно на шаге X?**  
Напишите номер шага и что видите на экране (или скрин) — разберём точечно.
