# Бесплатный хостинг 24/7 (без лимита Replit «30 дней»)

Для **Litematic Hub**: Node API + PostgreSQL + старый React UI.  
Пользователи из **России** открывают сайт **без VPN** (нужен хостинг с нормальным доступом из РФ, не обязательно «сервер только в РФ»).

Проект: `Lite-Structure-Parser\Lite-Structure-Parser` (Docker в `deploy/`).

---

**Вывод:** для «бесплатно и не 30 дней как Replit» смотри **российские PaaS** или **свой VPS + Docker**.

## Вариант 1 — RelaxDev (рекомендуется)

- Сайт: https://relaxdev.ru  
- PaaS в РФ, деплой из **GitHub**, бесплатный tier для пет-проектов (без привязки к 30 дням Replit).  
- Поддержка помогает с деплоем (пишут на сайте).

**Идея:** залить монорепо в GitHub (приватный репо можно) → подключить RelaxDev → указать Dockerfile из `deploy/Dockerfile` (context = корень монорепо).

**Переменные окружения:**

- `PORT` — как задаёт платформа  
- `DATABASE_URL` — Postgres (встроенный у RelaxDev или внешний)  
- `STATIC_DIR` — если образ как у нас: `/app/public`

После деплоя — **перезагрузить** все `.litematic`.

---

## Вариант 2 — Conhos (контейнеры + Postgres в одном месте)

- Документация: https://conhos.ru/docs/HostingNodePostgres.md  
- Российский контейнерный хостинг, Node + Postgres в одном `compose`-подобном конфиге.

Подходит, если RelaxDev не подойдёт по лимитам. Нужен репозиторий или загрузка проекта по их инструкции.

---

## Вариант 3 — Бесплатный VPS в Москве + Docker (максимум контроля)

Примеры: **GratisVPS** (Москва, free tier — проверь лимиты CPU/RAM на сайте), другие «free VPS Russia».

На VPS:

```bash
# Ubuntu, установить Docker
git clone <ваш-репозиторий>
cd Lite-Structure-Parser/deploy
cp .env.example .env
# задать POSTGRES_PASSWORD
docker compose up --build -d
```

Открыть порт **8080** в панели VPS.  
URL: `http://IP:8080` — из России обычно доступен.

Минусы: сам настраиваешь обновления и бэкапы БД.

## Вариант 4 — Render / Neon (не лучший выбор из РФ)

- **Render:** бесплатная PostgreSQL **истекает через 30 дней** — та же боль, что Replit.  
- Регистрация с российского IP часто **не проходит**.  
- Если уже есть аккаунт: web + Neon DB возможны, но для новых пользователей из РФ — ненадёжно.

`deploy\host-public-no-docker.bat` + Cloudflare Tunnel — бесплатно, но **не** «как хостинг»: ПК выключен → сайта нет.

## Минимальный план действий

1. Репозиторий GitHub с папкой `Lite-Structure-Parser` (уже с новым парсером в `api-server/src/lib/`).  
2. Зарегистрироваться на **RelaxDev** (или Conhos / free VPS).  
3. Деплой Docker-образа как в `deploy/Dockerfile`.  
4. Прописать `DATABASE_URL`, собрать, получить постоянный URL.  
5. Заново загрузить `.litematic` через сайт.

Я **не могу** создать аккаунт и нажать Deploy за вас — нужен ваш email/логин на выбранной платформе.
